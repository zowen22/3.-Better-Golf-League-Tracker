GOLF LEAGUE TRACKER — PROJECT HANDOFF DOC
Last updated: May 26, 2026

WHAT THIS IS
A self-hosted, multi-tenant golf league tracking web application. Built to run locally on a PC HDD initially, with plans to migrate to a NAS home server and eventually offer as a paid SaaS platform.

TECH STACK DECISIONS

Database: SQLite (file: golf_league.db)
Database GUI: DB Browser for SQLite
Backend: Python
Editor: VS Code
Version Control: Git
Future hosting: Home NAS → Cloudflare Tunnel
Future payments: Stripe
Domain registrar: Cloudflare Registrar

USER HIERARCHY
Three tiers:

Site Admin — full platform access, all leagues
League Admin — full access within own league
Member — read only within own league

Login System (current)

Login uses League ID + password
Admin uses same League ID with a different password
No individual user accounts yet — universal league passcode
Full account system designed and ready for future implementation


PLATFORM ARCHITECTURE
Multi-Tenant
Each league is fully isolated. Players are not shared across leagues. A user can be admin of one league and member of another via USER_LEAGUE_ROLES.
Course Management

Currently: manual entry, local to each league
Future Stage 2: site admin master course database
Future Stage 3: USGA course API integration
Schema is future-proofed: league_id = NULL + is_master_record = 1 = site-wide course

KEY DESIGN DECISIONS

Handicap method: Par-based differential (Gross - Par). NOT official WHS rating/slope method
Handicap calc: Last 4 rounds, drop highest, average remaining 3, multiply by 90%, cap at 18.0, negatives allowed, 2 round minimum
Scores carry across seasons: Yes by default
Skins: Per round, gross by default, opt-in per round, per-skin pot value split, carryover if no winner, must re-opt-in for carryover round
Match play: 2v2, A vs A and B vs B, A/B assigned fresh weekly by handicap, 1 point per hole + 1 overall per player = 20 points per matchup
Schedule: Round robin (avoid rematches) or manual, course assigned at build time, freely editable future rounds, warning on in-progress, locked on completed
Playoffs: Single elimination, default 4 teams, configurable, finals default 2 weeks
Bye weeks: No points awarded
Tiebreakers: Head to head → points percentage → all play record → scoring average (configurable)
Self reporting: Admin toggle, approval required toggle
All-play standings: Calculated at report time, not stored
Past round edits: Locked by default, site admin can unlock with 3-confirmation flow
Archive visibility: League admin controls member access
