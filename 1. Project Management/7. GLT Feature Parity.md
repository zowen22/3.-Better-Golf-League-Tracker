## Handicap Settings / Calculation

*(Reviewed 2026-07-04, once egress was opened — sources: [Recommended Handicap Settings](https://www.golfleaguetracker.com/glthome/cms/help/how-to/common-handicap-options), [Limiting Strokes per Hole for Handicapping](https://www.golfleaguetracker.com/glthome/cms/help/how-to/strokes-per-hole-limit))*

| GLT Setting / Option | BGLT Equivalent | Notes |
|---|---|---|
| Setting #70, "Number of Rounds Required To Establish a Handicap" (GLT recommends 2) | `league_settings.min_rounds_for_handicap` (default `2`) | Exact match on the recommended default. |
| Setting #71, "Number of Most Recent Rounds to Calculate Handicap" (GLT recommends 6) | `league_settings.rounds_to_average` (default `4`) | BGLT's default window is narrower (4 vs. GLT's recommended 6) — both are admin-configurable, not a capability gap, just a different starting default. |
| Setting #40, "Number of High Rounds to Throw Out" (GLT recommends `-33`, i.e. **33% of scores**, not a fixed count) | `league_settings.high_scores_to_drop` (default `1`, a fixed **count** of rounds) | **Real methodology difference.** GLT's high/low-drop settings are percentage-of-pool based (scales with how many rounds exist); BGLT's are a fixed round count regardless of pool size. Worth a product decision: keep BGLT's fixed-count model (simpler, predictable) or add a percentage-based option to match GLT exactly. |
| Setting #53, "Number of Low Rounds to Throw Out" (GLT recommends 0) | `league_settings.low_scores_to_drop` (default `0`) | Same fixed-count-vs-percentage caveat as above; default values match. |
| Setting #38, "Handicap Percent" (GLT recommends `1`, i.e. 100%) | `league_settings.handicap_percent` (default `90.0`) | Both configurable per-league; defaults differ (GLT recommends no reduction, BGLT defaults to the traditional 90% "playing handicap" convention) — not a gap, a different default. |
| Setting #36, "Maximum Calculated Handicap" (GLT recommends `54.0` for 18-hole leagues, `27.0` for 9-hole) | `league_settings.max_handicap_index` (season default `18.0`) + `platform_settings.max_handicap_index_ceiling` (site-wide ceiling, `54.0`) | BGLT's site-wide ceiling already matches GLT's 18-hole recommendation exactly; the per-season default (`18.0`) is far more conservative than GLT's recommended ceiling — admins can already raise it up to the platform ceiling, so this is a default-value difference, not a missing capability. No 9-hole-specific ceiling variant exists in BGLT today (GLT recommends 27.0 for 9-hole; BGLT doesn't distinguish 9 vs 18 for this setting). |
| Setting #112, "Handicap Calculation Type": **"2020 Rules (Custom)"** (course-difficulty/slope-adjusted) vs. **"Par"** (simple par-based, no course-handicap conversion) | `league_settings.diff_calculation_type` — `'par'` is fully implemented; `'whs'` (slope/rating) is a **dead dropdown option** (`admin/settings.html` offers it, but `handicap.py`'s engine always computes par-based differentials regardless of the selection) | **Directly resolves the open WP3.1 backlog question** ("implement real WHS slope/rating math, or remove the dropdown option?") — GLT's own docs confirm course-difficulty/slope-adjusted handicapping is a real, commonly-used mode (their own recommended default, in fact), not a niche USGA-only feature. This is a genuine parity gap, not just dead UI — leans toward "implement it" over "remove the option." |
| "Hole Stroke Limit" section — per-hole max score for handicap purposes, **"Net Double Bogey"** as the common preset, up to **6 different handicap-tiered ranges** (e.g., <10 hcp capped at 7, ≥10 hcp capped at 8), and an **automatic +5 adjustment when a player has no established handicap yet** | `league_settings.max_score_per_hole` (single global int, unset by default) + `max_score_action` (`'warn'`/block) | **Real gap.** BGLT has one global per-hole cap (or none) applied uniformly; GLT supports handicap-tiered ranges (different caps for different skill levels) and a "Net Double Bogey" (par + 2, adjusted for strokes received) preset rather than a raw fixed number. BGLT also has no equivalent of the +5 no-handicap penalty adjustment — a pre-eligibility player's hole-score cap isn't specially handled today. |
